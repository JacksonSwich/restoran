## 💾 4. БАЗА ДАННЫХ

### Конфигурация (config.py)

```python
DB_CONFIG = {
    "host": "localhost",
    "port": 3306,
    "user": "root",
    "password": "root",
    "database": "restaurant_management",
    "charset": "utf8mb4",
    "cursorclass": pymysql.cursors.DictCursor,
}
```

### Схема БД (по SQL-запросам в queries.py)

| Таблица | Назначение | Ключевые поля |
|---------|-----------|---------------|
| **categories** | Категории блюд | id, name, description, is_active |
| **dishes** | Блюда в меню | id, category_id, name, description, price, weight, cooking_time_minutes, is_available |
| **restaurant_tables** | Столики | id, table_number, seats_count, zone, status (free/occupied/reserved/out_of_service), order_id, amount |
| **customers** | Клиенты | id, full_name, phone, email, discount_percent |
| **orders** | Заказы | id, table_id, customer_id, status (new/cooking/ready/served/paid/cancelled), total_amount, final_amount, discount_amount, comment, created_at, closed_at |
| **order_items** | Позиции заказа | id, order_id, dish_id, quantity, price_at_order, comment |
| **payments** | Платежи | id, order_id, amount, payment_method (cash/card/online), status, paid_at, created_at |

### Представления (views)

| View | Назначение |
|------|-----------|
| **view_daily_revenue** | Ежедневная выручка |
| **view_popular_dishes** | Популярные блюда |
| **view_orders_full** | Полная информация по заказам |

### Слой доступа к данным (connection.py)

- `get_connection()` — новое соединение с БД
- `execute(query, params)` — INSERT/UPDATE/DELETE, возвращает rowcount
- `fetch_one(query, params)` — одна запись
- `fetch_all(query, params)` — все записи
- `execute_insert(query, params)` — INSERT с возвратом lastrowid

## ⚠️ 11. ЗАМЕЧАНИЯ И ТЕХНИЧЕСКИЙ ДОЛГ

1. **Нет обработки ошибок БД** — при недоступности MySQL приложение упадёт или покажет пустые данные
2. **Пул соединений отсутствует** — каждое соединение создаётся/закрывается на каждый запрос
3. **Огромный файл screens.py** (4743 строки) — критически нуждается в рефакторинге
4. **query.py использует сырые SQL** — нет ORM, нет миграций
5. **Пароль в открытом виде** — `root:root` в config.py
6. **Модели пустые** — `models/__init__.py` не содержит классов
7. **Нет тестов** — ни юнит-, ни интеграционных тестов
8. **Нет CI/CD** — нет GitHub Actions или других pipeline
9. **Коммит «Pushkin Commit»** — неинформативное название, не ясны изменения
10. **Неиспользуемые переменные и импорты** — в некоторых файлах есть мёртвый код
11. **Нет Docker-конфигурации** — сложно развернуть изолированно
